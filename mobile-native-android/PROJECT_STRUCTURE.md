# خريطة هيكلية مشروع إدهام (Project Structure Map) 🗺️

هذا الملف يوفر شرحاً تفصيلياً لكل المجلدات والملفات الرئيسية في المشروع لمساعدة المبرمجين على فهم "أين يجدون كل شيء".

## 1. المجلدات الرئيسية (Root Directories)
- **`app/`**: المجلد الأساسي لتطبيق الأندرويد (Kotlin).
- **`backend-api/`**: الكود المصدري للواجهة الخلفية (Backend) والخدمات.
- **`docs/`**: يحتوي على التقارير الفنية وخطط التطوير السابقة.
- **`disabled-files/`**: يحتوي على ملفات Java/Kotlin معطلة مؤقتاً أو تحتاج مراجعة.

---

## 2. تفاصيل تطبيق الأندرويد (`app/src/main/java/com/edham/logistics/`)

### 🏗️ الهيكلية البرمجية (Architecture)
يتبع التطبيق نمط **Clean Architecture** مع **MVVM**:

#### **A. حزمة الواجهات (`ui/`)**
- **`ui/home/`**: تحتوي على لوحات التحكم الرئيسية لكل دور (Driver, Accountant, Workshop, Supervisor).
  - **`accountant/`**: شاشات المحاسب (الفواتير، سندات القبض، كشوفات الحساب).
  - **`workshop/`**: شاشات الورشة (تتبع الصيانة، تتبع الزيت، إدارة قطع الغيار).
  - **`driver/`**: شاشات السائق (تسجيل الرحلات، إثبات التسليم، الاستبيانات).
- **`ui/screens/`**: شاشات مستقلة مثل الكاميرا الاحترافية (`EliteCameraActivity`).
- **`ui/viewmodel/`**: نماذج العرض العامة التي تربط البيانات بالواجهات.

#### **B. حزمة البيانات (`data/`)**
- **`data/local/`**: إدارة قاعدة البيانات المحلية (Room).
  - **`database/`**: تعريف الـ Database والـ Type Converters.
  - **`dao/`**: واجهات الوصول للبيانات (Queries).
  - **`entity/`**: جداول قاعدة البيانات (Vehicle, Trip, Maintenance, etc.).
- **`data/remote/`**: إدارة التواصل مع السيرفر.
  - **`api/`**: تعريفات Retrofit للـ Endpoints.
  - **`dto/`**: كائنات نقل البيانات (Data Transfer Objects).
  - **`mapper/`**: تحويل البيانات من الـ API إلى كائنات Domain.
- **`data/repository/`**: المستودعات التي تجمع بين البيانات المحلية والبعيدة.

#### **C. حزمة الصيانة (`maintenance/`)**
- **`MaintenanceManager.kt`**: "القلب النابض" لنظام الورشة، يدير الجدولة والتنبيهات.
- **`MaintenanceSchedulingEngine.kt`**: محرك ذكاء اصطناعي مصغر لجدولة الصيانة بناءً على حالة الأسطول.

#### **D. حزمة الخدمات والأساسيات (`core/`)**
- **`core/network/`**: إعدادات Retrofit، الـ Interceptors، ومراقبة سرعة الشبكة.
- **`core/auth/`**: إدارة الجلسات، التوكنات (Tokens)، وصلاحيات المستخدمين.
- **`core/utils/`**: دوال مساعدة عامة (تنسيق التاريخ، معالجة الصور).

---

## 3. تفاصيل الموارد (Resources - `app/src/main/res/`)
- **`layout/`**: ملفات الـ XML الخاصة بتصميم الشاشات.
- **`menu/`**: تعريفات القوائم الجانبية والسفلية.
- **`navigation/`**: ملف `nav_graph.xml` الذي يحدد مسارات التنقل بين الشاشات.
- **`values-ar/`**: النصوص المترجمة للغة العربية.

---

## 4. ملفات الإعدادات (Configuration Files)
- **`app/build.gradle.kts`**: إدارة المكتبات، إصدار التطبيق، ورابط السيرفر (`API_BASE_URL`).
- **`gradle.properties`**: إعدادات بيئة العمل (Memory, Proxy).
- **`HANDOVER_GUIDE.md`**: الدليل السريع للمبرمج المستلم.

---
**نصيحة:** إذا واجهت مشكلة في ملف معين، ابحث عن نسخة `.bak` في نفس المجلد، فقد تحتوي على الكود الأصلي قبل التعديلات.
