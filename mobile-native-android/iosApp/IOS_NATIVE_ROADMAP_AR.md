# خارطة الطريق الاحترافية - iOS Native (Swift)

هذا الملف هو مرجع التنفيذ الرسمي لنسخة الأيفون "إدهام" لضمان التطابق الكامل مع نسخة الأندرويد.

## الهدف
بناء تطبيق لوجستي iOS Native باستخدام **SwiftUI** و **Combine**، يتميز بالأداء العالي، الأمان، وسهولة الصيانة.

## حالة التنفيذ الحالية
- [x] **Sprint 0 - الأساس المعماري الأولي**
  - [x] إعداد هيكل المشروع (Core, UI, Network)
  - [x] إعداد `APIService` باستخدام Alamofire و Swift Concurrency
  - [x] إعداد نظام الألوان (Cockpit Theme) والخطوط
  - [x] بناء الـ ViewModel الأساسي لدعم نمط MVVM

## الخطة القادمة (Sprints)

### Sprint 1 - المصادقة والجلسات (Auth & Session)
- [ ] بناء شاشة الدخول (Login View) بتصميم SwiftUI عصري.
- [ ] إدارة الجلسة باستخدام **Keychain** لحفظ التوكن بشكل آمن.
- [ ] نظام التوجيه التلقائي (Auto-routing) حسب دور المستخدم (سائق، مشرف، إلخ).
- [ ] التعامل مع انتهاء الجلسة (Token Expiry).

### Sprint 2 - لوحة تحكم السائق المتكاملة (Driver Dashboard)
- [x] بناء واجهة الـ Cockpit الأساسية.
- [ ] إضافة التنبيهات الحية (Local Notifications).
- [ ] تطوير نظام الـ SOS والربط مع خدمة الطوارئ.
- [ ] بناء شاشات التقارير السريعة للسائق.

### Sprint 3 - إدارة الشحنات (Shipment Management)
- [ ] قائمة الشحنات (Shipment List) مع دعم الـ Pagination.
- [ ] تفاصيل الشحنة (Shipment Details) والخرائط المصغرة.
- [ ] إدارة حالات الشحنة (بدء، تحميل، تسليم) مع التحقق من الموقع.
- [ ] رفع المرفقات (صور الفواتير، التوقيع الرقمي).

### Sprint 4 - التتبع والخرائط (GPS & Maps)
- [ ] تكامل **CoreLocation** للتتبع في الخلفية.
- [ ] عرض الخرائط التفاعلية باستخدام **MapKit**.
- [ ] حساب المسافة والوقت المتوقع للوصول (ETA).
- [ ] إرسال إحداثيات السائق بشكل دوري للـ API.

### Sprint 5 - المشرف والتقارير (Supervisor & Analytics)
- [ ] بناء لوحة تحكم المشرف (Supervisor Dashboard).
- [ ] عرض الإحصائيات والرسوم البيانية (Swift Charts).
- [ ] إدارة الأسطول ومراقبة الحرارة عن بُعد.
- [ ] نظام تنبيهات المشرف عند حدوث انحرافات في درجات الحرارة.

## معايير الجودة
1. **الأداء:** استجابة الواجهة بسرعة 60 إطار في الثانية (60 FPS).
2. **الأمان:** تشفير كافة البيانات الحساسة المخزنة محلياً.
3. **التصميم:** الالتزام بدليل تصميم Apple (Human Interface Guidelines) مع الحفاظ على روح هوية "إدهام".
4. **الاختبار:** تغطية الـ Logic الأساسي باختبارات الوحدة (Unit Tests).
