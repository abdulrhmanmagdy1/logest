## 🎯 **المراحل الرئيسية - 5 مراحل أساسية**

### **المرحلة 1️⃣: إنشاء الطلب (Create Shipment)**

```
┌─────────────────────────────────────────────────────┐
│  📝 خطوة العميل: إنشاء شحنة جديدة                 │
├─────────────────────────────────────────────────────┤
│                                                     │
│  أ) تطبيق العميل (CreateShipmentFragment):        │
│     ├─ اضغط على "➕ شحنة جديدة"                   │
│     ├─ ملأ البيانات المطلوبة:                      │
│     │  ├─ موقع الاستقبال (Pickup)                │
│     │  ├─ موقع التسليم (Delivery)                │
│     │  ├─ تفاصيل البضاعة (Cargo Details)        │
│     │  └─ اختيار الخدمة (Service Type)           │
│     └─ اضغط "إنشاء الشحنة"                       │
│                                                     │
│  ب) ما يحدث في النظام:                            │
│     ├─ API: POST /api/v1/shipments                │
│     │  └─ التحقق من صحة البيانات                 │
│     ├─ توليد رقم تتبع فريد:                       │
│     │  └─ مثال: EDH-K5XYZ9ABC                    │
│     ├─ حساب السعر تلقائياً:                       │
│     │  ├─ السعر الأساسي: 100 SAR                │
│     │  ├─ سعر الوزن: 0.5 SAR/كجم                │
│     │  └─ الإجمالي: 125 SAR                     │
│     └─ حفظ الشحنة بحالة "pending"                │
│                                                     │
│  ج) النتيجة:                                       │
│     ✅ تم إنشاء الشحنة بنجاح                      │
│     ✅ رقم التتبع: EDH-K5XYZ9ABC                 │
│     ✅ السعر المقترح: 125 SAR                    │
│     ✅ حالة الشحنة: ⏳ معلقة (Pending)          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### **المرحلة 3️⃣: التأكيد والمعالجة (Admin Confirmation)**

```
┌─────────────────────────────────────────────────────┐
│  ✅ تأكيد الطلب من الإدارة                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  أ) لوحة تحكم المشرف:                              │
│     ├─ يرى الطلب الجديد بحالة "pending"            │
│     ├─ يراجع البيانات والسعر                       │
│     ├─ ينقر على "✅ تأكيد الطلب"                   │
│     └─ يختار: "تأكيد" أو "رفض مع السبب"           │
│                                                     │
│  ب) ما يحدث في النظام:                            │
│     ├─ API: PUT /api/v1/shipments/:id/status     │
│     │  {                                          │
│     │    "status": "confirmed",                   │
│     │    "confirmedBy": "admin_id",               │
│     │    "timestamp": "2026-05-22T10:05:00Z"     │
│     │  }                                          │
│     │                                             │
│     ├─ تحديث الحالة إلى "confirmed"              │
│     ├─ إضافة سجل في statusHistory                │
│     ├─ إرسال إشعار للعميل                        │
│     └─ بث التحديث عبر WebSocket                  │
│                                                     │
│  ج) النتيجة:                                       │
│     ✅ الطلب مؤكد والآن جاهز للتعيين              │
│     ✅ حالة الشحنة: 🟡 مؤكدة (Confirmed)        │
│     ✅ إشعار للعميل: "تم تأكيد طلبك"             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**API Details:**
```json
PUT /api/v1/shipments/507f1f77bcf86cd799439011/status

Request:
{
  "status": "confirmed",
  "notes": "تم التحقق من البيانات والسعر"
}

Response:
{
  "success": true,
  "shipment": {
    "trackingNumber": "EDH-K5XYZ9ABC",
    "status": "confirmed",
    "statusHistory": [
      {"status": "pending", "timestamp": "2026-05-22T10:00:00Z"},
      {"status": "confirmed", "timestamp": "2026-05-22T10:05:00Z"}
    ]
  }
}
```

### **المرحلة 5️⃣: تنفيذ الرحلة (Driver Execution)**

```
┌──────────────────────────────────────────────────────┐
│  🚚 تنفيذ الرحلة - سلسلة تحديثات الحالة           │
├──────────────────────────────────────────────────────┤
│                                                      │
│  تطبيق السائق (DriverDashboard):                   │
│  ├─ يرى الرحلة الجديدة المسندة                     │
│  ├─ ينقر "قبول الرحلة" → حالة = "accepted"      │
│  ├─ ينقر "بدء الرحلة" → يبدأ التتبع              │
│  └─ يحدّث الحالة مع كل مرحلة                      │
│                                                      │
│  سلسلة تحديثات الحالة:                             │
│                                                      │
│  1️⃣ At Pickup (في موقع الاستقبال)              │
│     ├─ الوقت: 10:10                              │
│     ├─ الإجراء: وصل موقع الاستقبال              │
│     ├─ التحديث: PUT /api/v1/shipments/:id/status│
│     │  {                                          │
│     │    "status": "at_pickup",                  │
│     │    "location": {"lat": 24.7136, ...},     │
│     │    "timestamp": "2026-05-22T10:10:00Z"   │
│     │  }                                          │
│     └─ الإشعار: 📍 "السائق وصل موقع الاستقبال"  │
│                                                      │
│  2️⃣ Picked Up (تم الاستقبال)                     │
│     ├─ الوقت: 10:15                              │
│     ├─ الإجراء: التقط البضاعة                     │
│     ├─ الإثبات: صورة + توقيع                     │
│     ├─ التحديث: PUT /api/v1/shipments/:id/status│
│     │  {                                          │
│     │    "status": "picked_up",                  │
│     │    "proofOfPickup": {                      │
│     │      "images": ["url1", "url2"],           │
│     │      "signature": "base64",                │
│     │      "notes": "تم الاستلام بسلام"         │
│     │    }                                        │
│     │  }                                          │
│     └─ الإشعار: ✅ "تم استقبال البضاعة"        │
│                                                      │
│  3️⃣ On The Way (في الطريق)                       │
│     ├─ الوقت: 10:20 - 10:50 (مستمر)             │
│     ├─ الإجراء: في الطريق نحو التسليم             │
│     ├─ GPS Updates: كل 5-30 ثانية                │
│     ├─ التحديث: PUT /api/v1/drivers/:id/location│
│     │  {                                          │
│     │    "location": {                            │
│     │      "lat": 24.7250,                       │
│     │      "lng": 46.6800,                       │
│     │      "speed": 95,                          │
│     │      "heading": 245                        │
│     │    },                                       │
│     │    "timestamp": "2026-05-22T10:25:00Z"   │
│     │  }                                          │
│     └─ الإشعار: 🟡 "في الطريق - متبقي 20 دقيقة"│
│                                                      │
│  4️⃣ Arriving Soon (الاقتراب)                      │
│     ├─ الوقت: 10:45                              │
│     ├─ الإجراء: في محيط 2 كم من الموقع          │
│     ├─ التحديث: PUT /api/v1/shipments/:id/status│
│     │  {                                          │
│     │    "status": "arriving_soon",              │
│     │    "distanceRemaining": 2                  │
│     │  }                                          │
│     └─ الإشعار: 📍 "السائق سيصل في 5 دقائق"   │
│                                                      │
│  5️⃣ At Delivery (في موقع التسليم)              │
│     ├─ الوقت: 10:50                              │
│     ├─ الإجراء: وصل موقع التسليم                │
│     ├─ التحديث: PUT /api/v1/shipments/:id/status│
│     │  {                                          │
│     │    "status": "at_delivery",                │
│     │    "location": {"lat": 21.5169, ...}     │
│     │  }                                          │
│     └─ الإشعار: 📍 "السائق وصل موقع التسليم"   │
│                                                      │

│     │      "timestamp": "2026-05-22T11:00:00Z" │
│     │    }                                        │
│     │  }                                          │
│     └─ الإشعار: ✅ "تم تسليم البضاعة بنجاح"   │
│                                                      │
│  7️⃣ Completed (مكتملة)                           │

│     ├─ الإجراء: إنهاء الرحلة وتسجيل التقييم     │
│     ├─ الأرباح: 150 SAR (محسوبة تلقائياً)       │
│     ├─ التحديث: PUT /api/v1/shipments/:id/status│
│     │  {                                          │
│     │    "status": "completed",                  │
│     │    "completionDetails": {                  │
│     │      "totalTime": 0.92,                    │

│     │      "earnings": 150,                      │
│     │      "rating": null                        │
│     │    }                                        │
│     │  }                                          │
│     └─ الإشعار: 🎉 "الرحلة مكتملة - قيّم الخدمة"│
│                                                      │
│  📊 ملخص الرحلة:                                    │
│  ├─ الوقت الكلي: 55 دقيقة                         │
│  ├─ المسافة: 450 كم                             │
```
