⚠️
Socket.IO / WebSocket: Requires Configuration
WebSockets must be explicitly enabled in Azure Portal. A 30-minute idle timeout applies (configurable). A web.config entry is required to disable the IIS WebSocket module for Socket.IO to work correctly.
⚖️ Pros & Cons
✅ Best-in-class enterprise compliance (SOC 2, ISO 27001, HIPAA, ZATCA/Saudi compliance ready)
✅ Native CI/CD with Azure DevOps; blue-green deployment slots on P-tier
✅ Azure Key Vault + Managed Identity — zero secrets in code or env vars
✅ Cosmos DB MongoDB API requires zero code changes — drop-in replacement
✅ 99.95% SLA on App Service; 99.999% on Cosmos DB
⚠️ WebSocket config required — web.config must disable IIS WebSocket module
⚠️ 30-min idle timeout on WebSocket connections
❌ Most expensive option at ~$195/mo; unpredictable Cosmos DB billing under heavy load
❌ Next.js on Static Web Apps has limited SSR support vs Vercel or self-hosted
❌ Azure learning curve — portal, CLI, IAM policies are complex for small teams

⚠️
Socket.IO / WebSocket: ALB Sticky Sessions Required
Requires an Application Load Balancer with sticky sessions enabled. ALB has a 60-second idle timeout by default — must be increased. Without stickiness, Socket.IO fails across multiple EB instances.
⚖️ Pros & Cons
✅ Largest service catalog — S3, SES, SNS, SQS, Lambda all available with zero config
✅ AWS Amplify SSR natively supports Next.js 14 with automatic Lambda@Edge deployment
✅ MongoDB Atlas M10 on AWS: low-latency co-location in same region
✅ AWS Secrets Manager + IAM Roles — robust zero-trust secret management
✅ EC2 Reserved Instances reduce compute cost by 40–60%
⚠️ ALB required for WebSocket sticky sessions — adds $18/mo and config complexity
⚠️ Amplify SSR has a 1 MB response limit and 25-distribution quota per account
⚠️ ElastiCache Extended Support surcharge since Feb 2026 (migrate to Valkey)
❌ Elastic Beanstalk is dated; AWS recommends ECS/Fargate for modern workloads
❌ AWS billing is complex — egress charges, AZ data transfer, and hidden fees accumulate
❌ Steepest learning curve; IAM misconfiguration is a common security pitfall

❌ No managed database backups — must script your own mongodump + S3/B2 uploads
❌ Scaling requires manual VPS resize + migration — no auto-scaling

🏆 Architecture Recommendations by Use Case

Best For
🏢 Enterprise / Government / Saudi Compliance
Choose Azure if Edham Logistics must meet Saudi ZATCA, PDPL, or enterprise SLAs. Azure has UAE North and KSA data centers. Key Vault + Managed Identity eliminates secrets exposure. 99.999% Cosmos DB SLA is difficult to match elsewhere.

Best For
⚡ High-Scale / AWS Services Already Used
Choose AWS if you need SES for email, SNS/SQS for push notifications, or Rekognition for document scanning. Atlas M10 co-located in AWS us-east-1 gives the lowest latency to your backend.
