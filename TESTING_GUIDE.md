# 🧪 اختبار النظام - Testing Guide

## كيف تختبر النظام الكامل من البداية للنهاية

### **المتطلبات:**
- ✅ Backend server جاري على `http://localhost:5000`
- ✅ تطبيق Android جاهز أو محاكي Android
- ✅ 3 حسابات نشطة:
  - حساب عميل
  - حساب مشرف/إدارة

## 🧪 Test Case 1: إنشاء شحنة من العميل

### **الخطوات:**

**الخطوة 1:** فتح التطبيق بحساب عميل
```
1. فتح التطبيق
2. تسجيل الدخول ببيانات العميل
3. انتقل إلى "Create Shipment"
```

**الخطوة 2:** ملء بيانات الشحنة
```
Pickup Information:
  - Street: طريق الملك فهد
  - City: الرياض
  - Region: منطقة وسط
  - GPS: 24.7136, 46.6753

Delivery Information:
  - Street: شارع التقدم
  - City: جدة
  - Region: منطقة غرب
  - GPS: 21.5433, 39.1728

Cargo Details:
  - Type: General
  - Weight: 50 kg
  - Description: "أجهزة إلكترونية"
  - Quantity: 1
  - Fragile: Yes
```

**الخطوة 3:** اضغط "Create"
```
توقع:
✅ رسالة نجاح مع رقم تتبع
✅ مثال: "EDH-K5XYZ9ABC"
✅ إشعار في الهاتف: "تم إنشاء الطلب"
✅ السعر المحسوب تلقائياً
```

## 🧪 Test Case 5: تحديثات الحالة في الطريق

### **السيناريو:**

**عند الوصول لنقطة الاستلام:**
```
الخطوة 1: السائق ينقر "Arrived at Pickup"
  API: PUT /api/v1/shipments/id/status
  Body: { status: "at_pickup", location: {...} }
  
توقع:
✅ الحالة تتغير إلى "at_pickup"
✅ إشعار للعميل: "السائق في الطريق إليك"
✅ الموقع يحدّث على الخريطة
```

**عند استلام البضاعة:**
```
الخطوة 2: السائق ينقر "Pickup"
  - يلتقط توقيع العميل (Digital Signature)
  - يلتقط صورة للبضاعة
  
API: PUT /api/v1/shipments/id/status
Body: {
  status: "picked_up",
  location: {...},
  signature: "...base64image...",
  photo: "...base64image..."
}

توقع:
✅ الحالة: "picked_up"
✅ إشعار: "تم استلام الشحنة"
```

**في الطريق:**
```
الخطوة 3: تحديثات مستمرة أثناء الرحلة
  - تحديث الموقع كل 30 ثانية
  - حساب ETA ديناميكي
  - عرض الطريق على الخريطة
  
إشعارات:
✅ "في الطريق إلى جدة"
✅ "وقت الوصول: 1 ساعة من الآن"
```

**عند الوصول للتسليم:**
```
الخطوة 4: السائق ينقر "Arrived at Delivery"
  API: PUT /api/v1/shipments/id/status
  Body: { status: "at_delivery", location: {...} }
  
توقع:
✅ إشعار للعميل: "السائق وصل لديك"
✅ إمكانية الاتصال المباشر
```

**عند التسليم:**
```
الخطوة 5: السائق ينقر "Deliver"
  - يلتقط توقيع المستقبل
  - يلتقط صورة البضاعة (Proof)
  
API: PUT /api/v1/shipments/id/status
Body: {
  status: "delivered",
  location: {...},
  signature: "...base64image...",
  photo: "...base64image...",
  deliveryTime: "2026-05-22T14:30:00Z"
}

توقع:
✅ الحالة: "delivered"
✅ إشعار فوري للعميل: "تم تسليم الشحنة"
✅ الطلب منتقل للعميل لإكماله
```

## 🐛 Debugging Tips

### **إذا لم ترسل الإشعارات:**
```
1. تحقق من اتصال WebSocket
2. تحقق من نموذج الإشعارات في قاعدة البيانات
3. تحقق من صلاحيات المستخدم
```

### **إذا لم يتم حفظ الموقع:**
```
1. تحقق من صلاحيات الموقع في Android
2. تحقق من الإحداثيات المرسلة
3. تحقق من نموذج Shipment locationHistory
```

### **إذا لم تتغير الحالة:**
```
1. تحقق من صلاحيات المستخدم (يجب أن يكون مشرف أو سائق)
2. تحقق من صحة معرف الشحنة
3. تحقق من سجل الأخطاء (logs) على الخادم
```

### **إذا لم تصل الإشعارات:**
```
1. تحقق من اتصال قاعدة البيانات
2. تحقق من نموذج User لتأكد من معرف المستخدم
3. تحقق من خدمة الإشعارات في البرامج
```
